# Core Concepts

## Git and repo setup
- **Repository:** The main home for the project files and change history.
- **Commit:** A saved checkpoint of work.
- **Remote:** The GitHub copy of the project that the local repo connects to.
- **SSH:** A secure authentication method for connecting my machine to GitHub.
- **README:** The front-page explanation of the project.

## Azure
- **Azure account:** My access point for using Azure services.
- **Azure:** Microsoft’s cloud platform for creating and managing services over the internet instead of running everything on my own machine.
- **Subscription:** The top-level billing and management boundary for Azure resources.
- **Resource group:** A logical container that keeps related Azure resources together so they can be managed as one unit.
- **Storage account:** The Azure resource that provides access to storage services such as blobs, files, queues, and tables.
- **Blob storage:** A storage service for files and other unstructured data in Azure.
- **Role-Based Access Control (RBAC):** Azure’s access control system for deciding who can do what on which resources.
- **Azure Data Factory (ADF):** Azure’s cloud service for building and running data movement and data workflow pipelines.
- **Azure Databricks:** An Apache Spark-based analytics service in Azure used for large-scale data processing, transformation, and analysis.
